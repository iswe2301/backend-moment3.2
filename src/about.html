OM SIDAN

Uppgiftsbeskrivning

Denna webbaplikation har tagits fram i kursen Backend-baserad webbutveckling (DT207G) med syftet att att erbjuda en plattform där användare kan dela med sig av sina jobberfarenheter. Applikationen möjliggör för användarna att lägga till nya erfarenheter, visa befintliga, redigera detaljer om sina upplevelser och även radera dem när de inte längre är aktuella.

Applikationen bygger på en kombination av Node.js och Express för serverhantering, medan MongoDB har använts som databasserver för att lagra data om erfarenheterna.

Slutsatser

Projektet har varit både kul och lärorikt, särskilt erfarenheten av att använda MongoDB som databasserver.

Jag upplevde att användningen av en NoSQL-databas förenklade utvecklingsprocessen, mycket tack vare möjligheten att definiera och validera data genom scheman. Jag upplevde att koden blev enklare att förstå och att det även blev mindre kod i själva routsen.

Jag har hostat min databas via MongoDB Atlas och min webbtjänst via Render. Jag upplevde detta som smidigt då det bara var att använda en anslutningssträng som en miljövariabel hos Render för att kunna koppla samman webbtjänsten med databasen.

För att effektivisera utvecklingen valde jag att klona mitt projekt från tidigare moment (moment 2.2) och återanvända befintlig kod. Med bara några få justeringar så fungerade applikationen precis som i moment 2. Det var verkligen smidigt och det enda som behövde justeras var att ändra så att id stämde överrens med hur MongoDB anger id, samt att justera egenskaperna i mina objekt så att de överrensstämne namnmässigt med mitt Mongoose-schema.
